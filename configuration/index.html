    # appID,token

    频道BOT的登陆凭证，前往QQ机器人管理端 (opens new window)注册申请官方BOT

    # sandbox

    沙箱环境，开启这个会在你机器人管理端设置的频道中主动消息无限制，具体查看腾讯官方文档：QQ机器人文档 (opens new window)

    监听消息类型，建议留空开启全部消息类型，本项目只用到了频道信息和私信信息

    # area

    机器人类型，分为公域（public）或者私域（private），在你创建BOT时候自己选择

    私域机器人只能在自己的频道使用，公域可以拉近别人频道使用

    # master

    你自己的频道账号ID，这个可以给BOT发送消息，日志终端会打印出来，错填会影响部分功能

    # header

    指令开头的字符，频道官方推荐为 /

    # countThreshold

    用户在一小时使用指令的次数的阈值，按整点计算，如 13:00~14:00 。如果用户在过去一小时内使用指令的次数超过了该值，BOT 会向持有者发送私聊信息，提示所有超量使用的用户和使用次数，以便持有者对超量使用指令的用户进行处理。

    # helpMessageStyle

    帮助信息样式，可以设置为 message, embed,ark,card

    • message 样式为所有指令以单条消息直接发送。
    • embed 样式为每条指令为单条消息，与message类似。
    • card 样式为所有指令以图片形式发送
    • ark样式与embed类似（官方逐渐舍弃，公域）

    # logLevel

    日志输出等级。

    # dbPort

    数据库端口。注意，Docker 启动修改此值时，需将 redis.conf 中的 port 修改为与此处相同的值。

    # webconsole

    Web Console 功能相关配置

    # enable

    是否启用 Web Console 功能，true 表示启用。

    # consolePort

    Web Console 页面端口，默认情况下为 80，以便直接通过服务器 IP 或 localhost 进行访问，可以自行配置。注意，Docker 启动修改此值时，需将 services.bot.ports 的第一个数字修改为与此处相同的值。

    # tcpLoggerPort

    log4js 日志输出端口。

    # jwtSecret

    JWT 验证秘钥，不提供默认配置，重要，可以随意输入长度为 6~16 的仅由字母和数字组成的字符串，最好不要有特殊含义。

    # adminName

    网页控制台的登录名

    # adminPwd

    网页控制台的登陆密码，需要开启网页控制台才生效

    # qiniu

    配置图片上传对象存储，需要自行去注册：七牛云官网 (opens new window)

    用户每人都有10G存储空间和10G CDN流量，给BOT用足够了

    # QAccessKey

    ·七牛云官网给的资源访问密钥

    # QSecretKey

    七牛云官网给的资源访问密钥

    # CloudUrl

    图片上传到对象存储的访问地址，也就是你给对象存储设置的CDN加速域名

    注意不要将域名绑定到源站，不然流量额外收费

    # Bucket

    图片上传到对象存储的位置，建议类型设置为公有，不然还得自己带密钥访问

    # cookies.yml

    genshin 插件用于执行 mys, uid, aby 等指令所需的 cookies ，可配置多个。

    index: 0
cookies:
  - cookieA
  - cookieB

    # commands.yml

    该文件将会在 BOT 初次运行后生成，所以你并不能在启动 BOT 前配置指令。

    # 基本结构

    Adachi-GBOT 中的指令被分为三类 order, switch 和 enquire ，你需要了解他们的共性和特性。每个指令都以 cmdKey 为顶级标识，你可以通过使用 #help -k 指令（或根据你的配置）来查看每个指令对应的 key 。每种指令都有四个公共配置字段：

    # type

    分为 order, switch, enquire ，不可修改。

    # auth

    最低操作权限，可设置为1, 2, 3, 4, 5：

    • 1 表示除封禁用户外所有人可使用
    • 2 表示当前频道管理员及以上 可使用，管理员需要频道主指定
    • 3 表示当前频道主可使用，默认频道主有自己频道的管理权限
    • 4表示全局管理员可使用，由机器人拥有者指定
    • 5表示只能机器人拥有者使用

    # scope

    指令使用位置，可设置为 1, 2, 3：

    • 1 表示仅群聊可使用
    • 2 表示仅私聊可使用
    • 3 表示群聊和私聊均可使用

    # enable

    是否启用指令。

    # Order 类指令

    Order 具有唯一可配置项 headers ，表示指令头，为数组类型，可以设置多个。默认情况下，指令头前会被添加上 setting.yml 中的 header 的值，如果你不想在某条指令加上它，可以在指令头前加上双下划线 __ 。如:

    silvery-star.wish:
  type: order
  auth: 1
  scope: 3
  headers:
    - __wish
    - w
  enable: true

    这将为祈愿十连导出两个指令，wish /w ，当然，你也可以配置更多。

    # Switch 类指令

    Switch 拥有四个可配置项，header, onKey, offKey 和 mode。

    • mode 开关模式，分为 single 和 divided ，它们的作用将在下面阐述
    • header 指令头，只可配置一个，当 mode 设置为 divided 无效，此处同样可以通过双下划线来屏蔽 setting 中配置的 header
    • onKey/offKey 表示开/关的关键词

    single 模式表示单指令头，使用关键词区分开/关状态，如：

    # setting.yml => header: "#"
silvery-star.alias-customize:
  type: switch
  auth: 2
  scope: 3
  mode: single
  onKey: add
  offKey: rem
  header: alias
  enable: true
# 导出指令 #alias [add|rem] [本名] [别名]

    divided 模式表示拆分指令头，使用指令头区分开/关状态，如：

    # setting.yml => header: ""
silvery-star.alias-customize:
  type: switch
  auth: 2
  scope: 3
  mode: divided
  onKey: 增加别名
  offKey: 删除别名
  header: ""
  enable: true
# 导出指令 1. 增加别名 [本名] [别名]  2. 删除别名 [本名] [别名]

    # Enquire 类指令

    Enquire 类指令只有一个单独的可配置项 sentences ，表示 “询问句式”。在一个句子中，包含由 #{} 包裹的关键字段和其余部分，在配置时，在保证保留所有的关键字段的前提下，你可以随意的打乱句式的顺序，指令都能正常的运行，如：

    enquire.example:
  type: enquire
  auth: 1
  scope: 3
  sentences:
    - "告诉我#{CHARACTER}的元素属性"
    - "#{CHARACTER}的元素属性是什么"
  enable: true

    # genshin.yml

    genshin 插件相关配置。

    # 默认配置

    cardWeaponStyle: normal
cardProfile: user
serverPort: 8080
showCharScore: true
wishLimitNum: 99

    # cardWeaponStyle

    用户信息查询卡片武器显示样式，可选 normal，weaponA 和 weaponB 。图例在依次在下方。

    # cardProfile

    用户信息查询卡片头像显示，可选 random 和 user 。设置为 random 时，头像会从玩家所用于的角色中随机抽取；设置为 user 时，头像为查询用户的 QQ 头像。

    # serverPort

    genshin 插件的后端端口

    # showCharScore

    是否显示评分

    # wishLimitNum

    模拟抽卡一次性最大次数
